Explore adaptive optimization methods whose core DNA is a mathematically explicit adaptive metric, preconditioner, or rescaling rule.

Mathematical DNA to build from:
- Diagonal preconditioning: scale each coordinate by a statistic such as `1 / sqrt(G_t + eps)` or `1 / sqrt(v_t + eps)`, where `G_t` is cumulative squared gradients and `v_t` is an EMA of squared gradients.
- Online geometry / mirror-descent view: interpret adaptive methods as gradient descent in a time-varying diagonal metric chosen from past gradients.
- First-moment and second-moment estimation: use `m_t` and `v_t` as explicit stochastic estimators, not just heuristics; reason about bias correction, lag, and estimator variance.
- Decoupled regularization: separate weight decay from the adaptive gradient transform instead of hiding it inside the preconditioned update.
- Trust-ratio scaling: normalize the step using parameter norm or block norm so the method behaves consistently across layers with different scales.

Promising formal directions:
- AdaGrad-family methods where cumulative curvature estimates naturally shrink aggressive coordinates.
- Adam / AdamW / RMSprop variants with better control of estimator drift, bias correction, or heavy-tailed noise.
- Adaptive methods with explicit clipping or norm control before the adaptive rescaling.
- Blockwise or layerwise adaptive metrics that approximate richer geometry while staying cheaper than full matrix methods.
- Adaptive epsilon, adaptive decay rates, or signal-to-noise-aware rescaling rules justified by the statistics being tracked.

Reality check:
- This island has real theory behind it, especially for AdaGrad-style geometry and convergence of carefully defined Adam variants.
- But many successful practical tricks here are still partly empirical: warmup design, beta schedules, and many Adam-family tweaks often work because they stabilize training, not because there is a clean theorem behind every choice.

Aim for methods that get the fast early progress adaptive optimizers are known for without sacrificing long-horizon stability, numerical safety, or robustness to sparse gradients and wildly different layer scales.

Formal principles and highlights:
- Canonical AdaGrad update:
  `G_t = sum_{i=1}^t g_i \odot g_i`
  `x_{t+1} = x_t - eta * g_t / sqrt(G_t + eps)`
  The key insight is coordinate-wise self-normalization: coordinates that have seen large gradients get smaller future steps.
- Canonical Adam / AdamW update:
  `m_t = beta1 * m_{t-1} + (1 - beta1) * g_t`
  `v_t = beta2 * v_{t-1} + (1 - beta2) * (g_t \odot g_t)`
  `mhat_t = m_t / (1 - beta1^t)`
  `vhat_t = v_t / (1 - beta2^t)`
  `x_{t+1} = x_t - eta * mhat_t / (sqrt(vhat_t) + eps)`
  AdamW adds decoupled decay: `x_{t+1} <- x_{t+1} - eta * lambda * x_t`.
- AdaGrad highlight: the original adaptive-subgradient regret bound is coordinate-aware; the algorithm competes with a comparator measured in a data-dependent diagonal geometry rather than a single global step size.
- AMSGrad / Adam-convergence highlight: the core fix is to enforce a nondecreasing second-moment denominator, typically via `vhat_t = max(vhat_{t-1}, v_t)`, to avoid pathological oscillations in the effective learning rate.
- Mirror-descent insight: many adaptive methods can be re-read as online mirror descent with a time-varying quadratic regularizer `R_t(x) = 1/2 * x^T H_t x`, where `H_t` is diagonal or block-diagonal and learned from gradient history.
- Trust-ratio insight from LARS / LAMB-style methods:
  `Delta x_t = -eta * trust_ratio_t * p_t`
  `trust_ratio_t = ||x_t|| / (||p_t|| + eps)`
  This explicitly separates direction quality from magnitude calibration across layers.
- Heavy-tail insight: one strong modern explanation for why Adam-family methods can outperform vanilla SGD on language-model-style problems is that diagonal adaptivity can partially compensate for class- or coordinate-wise heavy-tailed gradient imbalance.
- Design prompt: when inventing a new method, be explicit about which statistic is being estimated, what bias/variance tradeoff that estimator has, and why the resulting metric should be better than a plain diagonal EMA.
