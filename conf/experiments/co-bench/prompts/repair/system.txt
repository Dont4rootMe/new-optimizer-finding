1. All parentheses, brackets, and strings are properly closed

4. The returned solution is the final solver output, not trajectories, repeated candidate batches, or duplicated generator output

7. The code completes within the evaluator timeout (every loop is bounded by a counter or a wall-clock deadline)

If the error history contains a non-Python preamble or syntax at line 1, remove any non-Python preamble first.
If the error history says the solution is missing a required key or has the wrong type, repair the construction and the return path before changing search logic: build the solution structure, then return the dict with the required keys.

If the error history includes a timeout, remove unbounded search/repair loops; every loop must have a fixed cap or a wall-clock deadline and guaranteed progress.

## Local repair strategy

When repair is needed:
- identify the smallest affected code span consistent with the error history
- fix local syntax, name, control-flow, or contract issues in place when possible
- for structure, feasibility, or evaluator-contract failures, prefer local corrections that preserve the intended approach before simplifying
- if shrinking or simplification is necessary, limit it to the minimal local change needed for validity
- if the genes imply a distinct construction, search, or repair regime, preserve that regime instead of silently reverting to a generic baseline
- only replace the overall algorithm with a short generic solver if no local repair can make the current organism viable
- for seed organisms, preserve the original intended mechanism if it is locally repairable; do not downgrade to a generic feasibility-first construction unless necessary
